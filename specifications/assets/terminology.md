# OracleToolkit OS Terminology Standards

## Purpose

This document standardizes platform terminology across UI, documentation, code, database design, and AI prompts.

## Product Terms

| Term | Use | Do Not Use |
|---|---|---|
| OracleToolkit OS | Full platform name | Toolkit, tool collection |
| Workspace | Project cockpit | Portal, launcher page |
| Application | Specialized capability inside OS | Tool, standalone utility |
| Project Memory | Connected implementation knowledge store | Notes, random memory |
| Timeline | Chronological implementation history | Activity feed only |
| Governance | RAID, decisions, gates, approvals | Admin checklist |
| Lifecycle | Phase-based implementation journey | Process list |
| Phase | Implementation stage | Step, tab |
| Gate | Formal stage review | Checkbox |
| Deliverable | Versioned implementation artifact | File only |
| Evidence | Proof supporting completion/status | Attachment only |

## Status Terms

Use these standard status values unless a specific entity requires more detail:

- Not Started
- In Progress
- Blocked
- Ready for Review
- Approved
- Deferred
- Rejected
- Complete
- Archived

## Versioning Terms

- Alpha: foundational capability, internal testing
- Beta: feature complete enough for broader review
- Release Candidate: production-ready pending final validation
- GA: stable public release
