# OracleToolkit OS Glossary

## Activity

A required unit of work inside an implementation phase. Activities have owner, status, due date, dependencies, and evidence.

## AI Advisor

The future intelligence layer that provides context-aware recommendations, readiness validation, impact analysis, and implementation guidance.

## Application

A specialized OracleToolkit capability that accelerates implementation work and writes structured outputs into Project Memory.

## BCEA

Budgetary Control and Encumbrance Accounting. In OracleToolkit, BCEA design includes control budgets, control levels, funds check behavior, budget exceptions, dependencies, and testing.

## Deliverable

A versioned artifact produced manually or by an application. Deliverables may contain many detailed memory records.

## Evidence

A link, file, note, timestamp, or record that proves a task, decision, gate, or deliverable status.

## Gate

A formal phase exit review based on required deliverables, decisions, evidence, risks, and readiness score.

## Governance

The management of risks, assumptions, issues, dependencies, decisions, change requests, approvals, and escalations.

## Implementation Lifecycle

The phase-based Oracle Cloud implementation journey from Mobilization through Hypercare.

## Phase

A major stage of the implementation lifecycle, such as Discovery, SIT, UAT, Deployment, or Hypercare.

## Project Memory

The persistent implementation knowledge layer that stores decisions, assumptions, risks, open items, deliverables, RICE objects, testing records, cutover records, and lessons learned.

## Readiness Score

A calculated or manually reviewed score indicating whether a phase, gate, module, or project is ready to proceed.

## RICE

Reports, Integrations, Conversions, Extensions, and Workflows.

## Timeline Event

An auditable chronological record of a significant implementation event, such as deliverable creation, phase gate approval, risk closure, or go-live.

## Workspace

The project cockpit where implementation users manage the active project, lifecycle, applications, deliverables, memory, governance, and timeline.
